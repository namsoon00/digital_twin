# Notification Terminology

알림 메시지와 설정 화면에서는 투자 모델의 판단값과 알림 발송 정책값을 분리해서 표현한다.

## 관계 판단

- 사용자 표시명: `관계 판단`
- 강도 표시명: `관계 신호`
- 목적: 종목을 매수 후보, 매도 후보, 보유 점검 대상으로 볼지 관계 규칙과 근거로 판단한다.
- 대표 값: `ontologyRelationContext.signalStrength`, `holdingDecisionScore`
- 대표 규칙: `ontologyRelationRules`
- 메시지 내용: 모델명, 엔진 버전, 성립 규칙, 근거, 부족 데이터, AI 프롬프트
- 금지 표현: 발송 기준처럼 보이는 `발송 점수`, `꿀점수`

기존 `buyScoreFormula`, `sellScoreFormula`, `profitTakeScoreFormula`, `lossCutScoreFormula`는 보조 근거 또는 과거 비교용이다. 새 알림 메시지에서 이 공식을 최종 판단 주체처럼 설명하지 않는다.

## 알림 발송

- 사용자 표시명: `알림 발송`
- 우선도 표시명: `발송 우선도`
- 목적: 이미 감지된 이벤트를 실제 텔레그램/웹 알림으로 보낼지 판단한다.
- 내부 호환 키: `honeyScore`, `honeyThreshold`, `honeyReasons`
- 대표 공식: `notificationScoreFormula`
- 메시지 내용: 발송 우선도, 발송 공식, 발송 대입값, 발송 부족 데이터, 발송 판정 내역
- 금지 표현: 투자 모델 점수처럼 보이는 `모델링 점수`, 혼합 헤더인 `점수 계산`, 별칭인 `꿀점수`

## Message Layout

왕초보·초보 알림은 다음 순서의 압축 구조를 사용한다.

1. `판단`: 대응, 쉬운 이유, 확인 필요 점수, 계정 기준을 표시한다.
2. `왜 알림이 왔나요?`: 새 변화와 발송 기준을 최대 3개로 표시한다.
3. `현재 상황`과 `밸류에이션`: 원천 수치와 적정가 범위·안전마진·근거 수준만 표시한다.
4. `핵심 근거`: TypeDB 추론 축을 최대 3개와 핵심 반대 신호 1개로 합쳐 표시한다.
5. `다음 조건`: 실행 조건, 판단을 바꿀 조건, 부족 데이터를 한 번만 표시한다.
6. `뉴스·공시 요약`: 신선도와 중요도가 높은 최대 2건의 제목, 일시, 출처, 본문 요약, 종목 영향을 표시한다.

`관계 분석으로 새로 확인한 사실`, `투자 판단 근거`, `전략 가이드`, `신뢰도와 부족 데이터`처럼 같은 사실을 반복하던 별도 구역은 왕초보·초보 메시지에서 사용하지 않는다. 중수·고수는 감사와 상세 분석을 위해 더 많은 근거를 볼 수 있다.

기사 요약은 제목을 요약으로 재사용하지 않는다. 본문 기반 한국어 요약이 없으면 `기사 본문 요약이 아직 준비되지 않았습니다`라고 표시하며, 같은 URL의 헤드라인·연구 근거 레코드는 병합해 가장 풍부한 본문 분석을 보존한다.

`발송 우선도`는 투자 판단을 뜻하지 않는다. 반복 억제, 장 시간 필터, 임계값, 유사 메시지 페널티가 반영되는 알림 운영 값이다.

AI 의견은 검증된 구조화 응답만 메시지에 반영한다. 연결된 공시/뉴스가 없으면 추정하지 않고 가격, 수급, 추세 중심으로 설명한다. 기사 AI 요약은 사건·주체·핵심 수치·종목 관련성을 완전한 한국어 문장으로 작성하고, 영향 판단과 분리한다. 프롬프트는 `aiPromptTemplates`에서 알림 타입별로 관리한다.
